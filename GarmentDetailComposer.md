# Garment Detail Composer 기획 문서

## 1. 개요

`Garment Detail Composer`는 하나의 스타일을 기준으로 원단, 머티리얼, 지퍼, 버튼, 버튼홀, 탑스티치, 그래픽 등의 요소를 2D 화면 위에서 조합하고 비교할 수 있는 웹앱이다.

이 서비스의 목적은 패션 디자인, 테크니컬 디자인, 소싱, MD가 분절된 표 데이터 대신 `옷처럼 보이는 시각 결과물`을 기준으로 빠르게 의사결정할 수 있도록 돕는 것이다.

## 2. 문제 정의

현재 스키마는 의류를 구성하는 다양한 요소를 잘 정의하고 있지만, 정보가 문서 단위로 흩어져 있기 때문에 아래 문제가 발생할 수 있다.

- 하나의 스타일이 어떤 소재와 부자재 조합으로 구성되는지 한 번에 보기 어렵다.
- 버튼, 지퍼, 탑스티치, 그래픽 변경이 실제 룩에 어떤 인상을 주는지 표만으로 판단하기 어렵다.
- 디자인팀, 테크팀, 소싱팀이 같은 데이터를 보더라도 머릿속에 떠올리는 결과 이미지가 다를 수 있다.
- 시즌 리뷰나 샘플 리뷰 때 조합 비교에 시간이 많이 든다.

## 3. 핵심 제안

이 서비스는 실제 3D 시뮬레이션이 아니라 `smart 2D visual composer`를 지향한다.

- 미리 준비된 의상 실루엣 템플릿을 사용한다.
- 각 부위에 소재, 색상, 그래픽, 부자재를 오버레이한다.
- 스티치, 버튼홀, 지퍼 등의 디테일은 라인, 마커, 반복 패턴 등으로 시각화한다.
- 결과적으로 사용자는 `완성복 사진`이 아니라 `의상처럼 보이는 디자인 시안 이미지`를 얻게 된다.

## 4. 핵심 가치

### 4-1. 빠른 조합 검토

원단이나 부자재를 교체했을 때 전체 인상이 어떻게 달라지는지 즉시 확인할 수 있다.

### 4-2. 팀 간 공통 시각 언어 제공

디자이너, 테크니컬 디자이너, MD, 소싱 담당자가 동일한 스타일 시안을 보면서 대화할 수 있다.

### 4-3. 리뷰 효율 향상

샘플 리뷰, 시즌 리뷰, 라인 리뷰 시 텍스트와 테이블보다 빠르게 비교할 수 있다.

### 4-4. 스키마 데이터의 실사용성 강화

지금의 데이터 사전을 단순 문서가 아니라 실제 서비스에서 활용 가능한 구조로 확장할 수 있다.

## 5. 대상 사용자

- 패션 디자이너
- 테크니컬 디자이너
- 소재 및 부자재 소싱 담당자
- MD 및 상품기획 담당자
- 시즌 리뷰를 진행하는 리더십

## 6. 사용자 시나리오

### 시나리오 A. 스타일 조합 검토

1. 사용자가 특정 `Style`을 연다.
2. 앞판, 뒷판, 디테일 뷰가 보인다.
3. 몸판에는 `Fabric`과 `Material`을 적용한다.
4. 앞여밈에는 `Button` 또는 `Zipper`를 적용한다.
5. 포켓이나 가슴 위치에는 `Graphic`을 적용한다.
6. 가장자리나 절개선에는 `TopStitch`를 적용한다.
7. 사용자는 두 가지 이상의 옵션을 나란히 비교한다.

### 시나리오 B. 컬러웨이 비교

1. 사용자가 하나의 `Style` 아래 여러 `Article`을 선택한다.
2. 컬러웨이별 소재 및 부자재 차이를 한 화면에서 본다.
3. 어떤 컬러웨이가 시즌 무드에 더 적합한지 비교한다.

### 시나리오 C. 디테일 검토

1. 사용자가 지퍼, 버튼홀, 탑스티치 디테일만 확대해서 본다.
2. 사양 차이와 배치 차이를 시각적으로 확인한다.
3. 샘플 수정 포인트를 빠르게 정리한다.

## 7. 데이터 활용 방식

현재 리포지토리의 스키마는 아래와 같이 활용할 수 있다.

- `Style`: 기준 스타일과 메타 정보
- `Article`: 컬러웨이 및 상품 변형 정보
- `Fabric`: 원단 조성, 두께, 폭, 공급처, 물성
- `Material`: 색상, 질감, 광택, 투명도 등 표면 특성
- `Graphic`: 프린트/아트워크 정보
- `Zipper`: 지퍼 종류, 규격, 소재 파트
- `Button`: 버튼 규격 및 소재
- `ButtonHole`: 버튼홀 사양
- `TopStitch`, `SubTopStitch`: 스티치 구성과 디테일
- `Season`: 시즌 맥락과 스토리

## 8. 화면에서의 표현 방식

### 8-1. Fabric / Material

- 영역 마스크 기반 색상 및 텍스처 적용
- 광택, 거칠기, 투명도 등은 시각 효과로 단순화해 반영

### 8-2. Button

- 정해진 anchor 위치에 반복 배치
- 크기, 색상, 간격을 반영

### 8-3. ButtonHole

- 버튼 위치 옆에 선형 마커 또는 슬롯 형태로 표현

### 8-4. Zipper

- 중심선 또는 절개선 기준으로 길이와 폭을 반영한 그래픽 오버레이

### 8-5. TopStitch / SubTopStitch

- 절개선이나 엣지 라인에 점선, 이중선, 반복 스티치 패턴으로 표현

### 8-6. Graphic

- 정해진 부위에 이미지 또는 벡터 오버레이

## 9. MVP 범위

초기 버전은 `셔츠 또는 재킷 1개 카테고리`에 집중하는 것이 적절하다.

### MVP 포함 기능

- 스타일 템플릿 1~2종 지원
- 앞판/뒷판 뷰 제공
- Fabric/Material 적용
- Button 또는 Zipper 적용
- Graphic 배치
- TopStitch 표시
- Article 단위 비교
- 결과 시안 저장

### MVP 제외 기능

- 실제 3D 시뮬레이션
- 패턴 기반 자동 드레이프
- 정교한 물리 엔진
- 모든 의복 카테고리 동시 지원
- 생산용 완전 자동 Tech Pack 생성

## 10. 최소 추가 데이터 요구사항

현재 스키마만으로도 개념 검증은 가능하지만, 더 자연스러운 화면을 위해 아래 메타데이터가 추가되면 좋다.

### 10-1. Garment Template

- 어떤 의상 템플릿을 사용하는지
- 예: Shirt Front, Shirt Back, Jacket Front

### 10-2. Placement Anchor

- 각 요소를 어디에 놓을지 정의하는 위치 정보
- 예: center-front, chest-left, cuff, pocket-edge

### 10-3. View Layer

- 앞판, 뒷판, 디테일 뷰 중 어디에 노출되는지

### 10-4. Render Asset

- 버튼 아이콘, 지퍼 그래픽, 텍스처 이미지, 그래픽 원본 등 실제 렌더 자산 경로

### 10-5. Rule Set

- 특정 부자재가 어떤 카테고리의 템플릿에만 허용되는지에 대한 규칙

## 11. 정보 구조 제안

### 화면 1. Style Composer

- 중앙: 의상 캔버스
- 좌측: 스타일/아티클 선택
- 우측: Fabric, Material, Button, Zipper, Graphic, TopStitch 패널

### 화면 2. Detail Inspector

- 특정 디테일 확대 뷰
- 버튼홀, 탑스티치, 지퍼 등 세부 사양 확인

### 화면 3. Compare Board

- 옵션 A와 옵션 B를 나란히 비교
- 컬러웨이 또는 부자재 변경안 비교

## 12. 기대 효과

- 시즌 리뷰와 샘플 리뷰 속도 향상
- 변경안 커뮤니케이션 비용 절감
- 데이터 사전과 실무 화면의 연결
- 소재 및 부자재 선택의 시각적 설득력 강화

## 13. 현실적인 기대치

이 서비스는 `자동으로 새로운 패션 이미지를 생성하는 AI 툴`이 아니다.

대신 아래에 강점이 있다.

- 준비된 템플릿 위에서 빠르게 조합을 보여준다.
- 실제 의류처럼 인식되는 2D 시안을 만든다.
- 표 중심의 정보를 훨씬 직관적인 화면으로 바꾼다.

즉, 이 서비스의 목표는 `포토리얼한 결과`가 아니라 `빠르고 설득력 있는 디자인 검토용 시각화`이다.

## 14. 한 줄 정의

`Garment Detail Composer`는 스타일 기준의 소재, 부자재, 디테일 조합을 2D 시안으로 시각화해 의류 기획과 디자인 의사결정을 빠르게 만드는 웹앱이다.
